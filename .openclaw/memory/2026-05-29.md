- Stall if no getUpdates activity > DEFAULT_POLL_STALL_THRESHOLD_MS=120s (user-configurable, min 30s).
- On stall: mark transport dirty, log "Polling stall detected ... forcing restart", stop runner/bot, backoff restart (2s->30s, 1.8x).
- Also handles 409 getUpdates conflicts (duplicate poller using same token) with restart + hint.
- Isolated ingress (spool) has additional handler timeout logic that can force restarts.
- Status publisher sets "connected: false" / lastError on errors → "offline" in UI/gateway health.
- Recoverable network errors (common on Windows DNS/IPv6/firewall/VPN) + no custom network tuning → repeated cycles.
